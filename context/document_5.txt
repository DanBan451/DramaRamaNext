The app they were building was a chatbot that had multiple tools where when enabled, each tool would be a dataset which would be RAG’d through. Here was the process:

On the frontend NextJS application, there would be a chatbot. The user would type in a question and then enable a tool which would be associated with a dataset. They hit send, which would call an endpoint on a websocket-lambda application (FastAPI) that would process_message. This process_message does a LOT like figuring out which tools were enabled, RAGing through those datasets and pulling out the best ranked chunks, and then we’d stream the response back to the frontend and after the streaming was done we’d call the save message from another lambda service that is solely forgetting the existing messages, storing them, chat history, etc.
